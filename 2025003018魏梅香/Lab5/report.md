# ArtSpaceApp 开发报告
## 一、应用展示内容
### 1. 应用主题
ArtSpaceApp 是一款基于 **Jetpack Compose** 开发的艺术作品展示应用，核心功能为沉浸式展示经典艺术画作，支持用户自由切换查看不同作品，同时展示作品的完整信息，为用户提供简洁、优雅的艺术欣赏体验。

### 2. 展示作品数量
应用内置 **3幅** 文森特·梵高的经典油画作品，具体清单：
1. 《The Starry Night》（星月夜）- 1889年
2. 《Green Wheat Fields, Auvers》（奥维尔的绿色麦田）- 1890年
3. 《Harvest in Provence》（普罗旺斯的收获）- 1888年

### 3. 核心信息展示
每幅作品同步展示**图片、标题、艺术家、创作年份**四类核心信息，满足用户对艺术作品的基础认知需求。

## 二、界面结构说明
应用采用**模块化、分层级**的界面设计，将整体界面划分为三大核心区块，通过 Jetpack Compose 可组合项实现嵌套与布局，结构清晰且易于维护。

### 1. 整体布局容器
- 根容器：`Surface` + `Column`
  - `Surface` 作为基础背景容器，铺满全屏并应用主题背景色；
  - `Column` 垂直布局管理器，将三大区块垂直排列，设置**水平居中、垂直均匀分布**，适配不同屏幕尺寸。

### 2. 三大界面区块
#### （1）艺术作品展示区
- 核心功能：展示艺术画作，模拟真实画框效果
- 可组合项：`Card` + `Box` + `Image`
- 嵌套逻辑：
  1. `Card` 作为外层画框，实现阴影、边框、圆角效果；
  2. `Box` 作为图片容器，实现图片居中对齐；
  3. `Image` 加载本地资源图片，填充容器并预留内边距。

#### （2）作品信息展示区
- 核心功能：展示当前作品的标题、艺术家、年份
- 可组合项：`Card` + `Column` + `Text` + `Spacer`
- 嵌套逻辑：
  1. `Card` 作为信息卡片，设置浅灰色背景突出信息区；
  2. `Column` 垂直排列文字信息；
  3. `Text` 分别展示作品标题（加粗）、艺术家+年份（灰色）；
  4. `Spacer` 分隔两行文字，优化排版间距。

#### （3）导航控制区
- 核心功能：提供「上一个/下一个」按钮切换作品
- 可组合项：`Row` + `Button` + `Text` + `Spacer`
- 嵌套逻辑：
  1. `Row` 水平布局，将两个按钮居中排列；
  2. 两个 `Button` 分别实现 Previous/Next 功能；
  3. `Spacer` 分隔两个按钮，避免界面拥挤。

### 3. 核心可组合项总结
| 区块名称         | 核心可组合项                          |
| ---------------- | ------------------------------------- |
| 作品展示区       | Card、Box、Image                      |
| 信息展示区       | Card、Column、Text、Spacer             |
| 导航控制区       | Row、Button、Text、Spacer              |
| 整体布局         | Surface、Column                        |

## 三、Compose 状态管理实现
应用通过 **Jetpack Compose 状态管理机制** 控制当前展示的作品索引，实现作品的动态切换，核心实现逻辑如下：

### 1. 数据准备
定义 `Artwork` 数据类封装作品信息，创建包含3个作品的列表 `artworks`，通过索引区分不同作品。

### 2. 状态定义
```kotlin
var currentArtworkIndex by remember { mutableStateOf(0) }
```
- `remember`：保证配置变更（如屏幕旋转）时，状态不会丢失，持续持有数据；
- `mutableStateOf(0)`：创建可变状态，初始值为`0`（默认展示第一幅作品）；
- `by` 委托：简化状态读写，直接通过变量名修改/获取索引值。

### 3. 状态驱动UI
当 `currentArtworkIndex` 状态发生变化时，Compose 会**自动重组UI**，刷新所有依赖该状态的组件：
- 作品展示区加载对应索引的图片；
- 信息展示区更新对应索引的文字信息；
- 导航按钮的条件逻辑基于最新索引执行。

## 四、Next / Previous 按钮条件逻辑
两个按钮均通过**边界判断**实现循环切换效果，避免索引越界异常，具体逻辑如下：

### 1. Previous（上一个）按钮逻辑
```kotlin
onPrevious = {
    currentArtworkIndex = if (currentArtworkIndex == 0) {
        artworks.size - 1 // 当前是第一幅，切换到最后一幅
    } else {
        currentArtworkIndex - 1 // 否则索引减1
    }
}
```
- 触发条件：点击「Previous」按钮；
- 核心规则：如果当前是**第一个作品（索引=0）**，点击后跳转到**最后一个作品**；否则索引减1。

### 2. Next（下一个）按钮逻辑
```kotlin
onNext = {
    currentArtworkIndex = if (currentArtworkIndex == artworks.size - 1) {
        0 // 当前是最后一幅，切换到第一幅
    } else {
        currentArtworkIndex + 1 // 否则索引加1
    }
}
```
- 触发条件：点击「Next」按钮；
- 核心规则：如果当前是**最后一个作品（索引=总数-1）**，点击后跳转到**第一个作品**；否则索引加1。

### 3. 效果总结
两个按钮实现**无限循环切换**，用户可以无限制浏览所有作品，无需担心边界失效问题。

## 五、遇到的问题与解决过程
### 问题1：图片加载失败，显示资源错误
- **问题描述**：运行应用后，作品展示区无法显示图片，日志提示资源ID无效。
- **原因分析**：
  1. 本地图片资源未正确放置在 `res/drawable` 目录下；
  2. 代码中引用的资源名称（如`artwork_1`）与实际图片文件名不一致。
- **解决方案**：
  1. 将3张作品图片重命名为 `artwork_1.png`、`artwork_2.png`、`artwork_3.png`；
  2. 放入项目 `app/src/main/res/drawable` 文件夹；
  3. 同步项目后，重新运行应用，图片正常加载。

### 问题2：状态失效，点击按钮无反应
- **问题描述**：点击 Next/Previous 按钮，界面无任何变化，作品不切换。
- **原因分析**：初始代码未使用 `remember` 包裹状态，每次重组都会重置索引为初始值，状态无法持久化。
- **解决方案**：使用 `remember { mutableStateOf(0) }` 管理索引状态，保证状态在UI重组时保留，按钮点击后索引正常更新，界面自动刷新。

### 问题3：界面布局混乱，组件错位
- **问题描述**：按钮、卡片、图片排列混乱，未居中对齐，间距不一致。
- **原因分析**：未给 `Column`、`Row` 设置正确的排列参数，缺少修饰符（Modifier）控制对齐方式和内边距。
- **解决方案**：
  1. 给根 `Column` 设置 `horizontalAlignment = Alignment.CenterHorizontally` 和 `verticalArrangement = Arrangement.SpaceAround`；
  2. 给按钮容器 `Row` 设置水平居中；
  3. 添加 `padding` 和 `Spacer` 优化间距，最终界面整齐统一。

### 问题4：文字信息排版拥挤，可读性差
- **问题描述**：作品标题和艺术家文字紧贴在一起，视觉效果差。
- **原因分析**：未添加间距组件，文字直接垂直排列。
- **解决方案**：在两个 `Text` 之间添加 `Spacer(modifier = Modifier.height(4.dp))`，设置固定间距，提升信息展示的可读性。

## 六、总结
ArtSpaceApp 基于 Jetpack Compose 实现了艺术作品的展示与切换功能，通过**模块化可组合项**拆分界面、**状态管理**控制UI刷新、**条件判断**实现循环导航，完成了完整的应用开发。开发过程中解决了资源加载、状态管理、界面布局等常见问题，掌握了 Compose 核心开发思路，应用具备简洁、易用、稳定的特点。
